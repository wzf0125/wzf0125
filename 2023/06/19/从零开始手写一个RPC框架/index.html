从零开始手写一个RPC框架

RPC简介

RPC（Remote Procedure Call） 即远程过程调用，通过名字我们就能看出 RPC 关注的是远程调用而非本地调用。

为什么要 RPC ？

因为，两个不同的服务器上的服务提供的方法不在一个内存空间，所以，需要通过网络编程才能传递方法调用所需要的参数。并且，方法调用的结果也需要通过网络编程来接收。但是，如果我们自己手动网络编程来实现这个调用过程的话工作量是非常大的，因为，我们需要考虑底层传输方式（TCP 还是 UDP）、序列化方式等等方面。

RPC 能帮助我们做什么呢？

简单来说，通过 RPC 可以帮助我们调用远程计算机上某个服务的方法，这个过程就像调用本地方法一样简单。并且！我们不需要了解底层网络编程的具体细节。 举个例子：两个不同的服务 A、B 部署在两台不同的机器上，服务 A 如果想要调用服务 B 中的某个方法的话就可以通过 RPC 来做。

一言蔽之：RPC 的出现就是为了让你调用远程方法像调用本地方法一样简单。

RPC原理

整个 RPC 的 核心功能看作是下面 5 个部分实现的：

  1. 客户端（服务消费端） ：调用远程方法的一端。
  2. 客户端 Stub（桩） ： 这其实就是一代理类。代理类主要做的事情很简单，就是把你调用方法、类、方法参数等信息传递到服务端。
  3. 网络传输 ： 网络传输就是你要把你调用的方法的信息比如说参数啊这些东西传输到服务端，然后服务端执行完之后再把返回结果通过网络传输给你传输回来。网络传输的实现方式有很多种比如最基本的 Socket 或者性能以及封装更加优秀的 Netty（推荐）。
  4. 服务端 Stub（桩） ：这个桩就不是代理类了。我觉得理解为桩实际不太好，大家注意一下就好。这里的服务端 Stub 实际指的就是接收到客户端执行方法的请求后，去指定对应的方法然后返回结果给客户端的类。
  5. 服务端（服务提供端） ：提供远程方法的一端。
  1. 服务消费端（client）以本地调用的方式调用远程服务；
  2. 客户端 Stub（client stub） 接收到调用后负责将方法、参数等组装成能够进行网络传输的消息体（序列化）：RpcRequest；
  3. 客户端 Stub（client stub） 找到远程服务的地址，并将消息发送到服务提供端；
  4. 服务端 Stub（桩）收到消息将消息反序列化为 Java 对象: RpcRequest；
  5. 服务端 Stub（桩）根据RpcRequest中的类、方法、方法参数等信息调用本地的方法；
  6. 服务端 Stub（桩）得到方法执行结果并将组装成能够进行网络传输的消息体：RpcResponse（序列化）发送至消费方；
  7. 客户端 Stub（client stub）接收到消息并将消息反序列化为 Java 对象:RpcResponse ，这样也就得到了最终结果。over!

Socket网络通信实战

Socket(套接字 ) 是一个抽象概念，应用程序可以通过它发送或接收数据。在使用 Socket 进行网络通信的时候，通过 Socket 就可以让我们的数据在网络中传输。操作套接字的时候，和我们读写文件很像。套接字是 IP 地址与端口的组合，套接字 Socket=（IP 地址：端口号）。

要通过互联网进行通信，至少需要一对套接字： 运行于服务器端的 Server Socket。

Nerry

简介

Netty是一个基于NIO的client-server(客户端服务器框架)，使用它可以快速简单地开发网络应用程序。

极大简化了TCP和UDP套接字服务器等网络编程，并且性能以及安全性等很多方面甚至都要更好

支持多种协议，FTP，SMTP，HTTP以及各种二进制和基于文本的传统协议

Netty成功地找到了一种在不妥协可维护性和性能的情况下实现易于开发，性能，稳定性和灵活性的方法

Netty的特点

  1. 异步和事件驱动：Netty使用异步的、非阻塞的I/O模型，通过事件驱动机制处理网络操作。这种设计使得Netty能够高效地处理大量的并发连接，同时提供低延迟和高吞吐量。
  2. 抽象和灵活的API：Netty提供了一套抽象的、易于使用的API，使得开发者能够轻松地构建各种类型的网络应用程序，包括服务器、客户端、代理、Web服务等。它支持多种协议，如TCP、UDP、HTTP、WebSocket等。
  3. 高性能：Netty经过优化，具有出色的性能。它通过内存池、零拷贝技术、线程池和可定制的传输层等特性，提供了低延迟、高吞吐量和可扩展性。
  4. 安全性：Netty提供了各种安全性功能，如SSL/TLS支持、加密和解密、身份验证等，帮助保护网络应用程序的数据和通信安全。
  5. 完善的扩展性：Netty的设计允许开发者通过自定义的ChannelHandler来扩展框架的功能。它还提供了丰富的插件机制和扩展点，使得开发者可以根据自己的需求进行定制和扩展。
  6. 易于测试和调试：Netty提供了模拟测试的工具和便利，使得编写单元测试和集成测试变得简单。它还提供了丰富的调试功能，如日志记录、错误处理和事件跟踪等。
  7. 社区活跃和广泛应用：Netty拥有庞大的开发者社区，提供了大量的文档、教程和示例代码。它被广泛应用于许多知名的网络应用框架和系统，如Elasticsearch、Apache Cassandra、RocketMQ等。

应用场景

  • RPC框架的网络通信工具
  • HTTP服务器
  • 即时通许系统IM(Instant Messaging)
  • 消息推送系统

Netty的使用流程

  1. 添加Netty依赖：首先，在你的项目中添加Netty的依赖项。你可以使用构建工具（如Maven、Gradle）或手动下载JAR文件并添加到类路径中。

  2. 创建EventLoopGroup：创建一个EventLoopGroup，它负责处理I/O事件的线程池。根据你的需求，可以选择创建NioEventLoopGroup（基于NIO）或EpollEventLoopGroup（基于Linux的epoll机制）。

  3. 创建Bootstrap：创建一个Bootstrap实例，用于配置和引导你的网络应用程序。Bootstrap负责初始化和连接Channel。

  4. 配置Bootstrap：使用Bootstrap的方法配置你的网络应用程序。例如，设置EventLoopGroup、选择Channel类型、添加ChannelHandler等。

  5. 连接远程主机：通过调用Bootstrap的connect()方法连接到远程主机。你需要指定远程主机的地址和端口。

  6. 发送和接收数据：一旦连接建立，你可以通过获取Channel并使用它发送和接收数据。

  7. 关闭连接和资源：当你完成了所有操作，记得关闭连接和释放资源。关闭连接会触发一系列的事件处理和资源清理。

使用kryo序列化传输实战

kryo简介

Kryo是一个快速、高效的Java对象序列化库，它可以将Java对象转换为字节序列，以便在网络上传输、存储到磁盘或进行缓存。Kryo的设计目标是提供高性能和紧凑的序列化方式，以替代Java标准库中的序列化机制，从而提高序列化和反序列化的速度。

以下是一些Kryo的特点和优势：

  1. 高性能：Kryo使用了一系列优化技术，如缓存复用、无需写入类描述符和字段名称等，以达到更高的序列化和反序列化性能。
  2. 紧凑的数据格式：Kryo生成的字节序列通常比Java标准库的序列化结果更小，这意味着它可以减少网络传输和存储所需的带宽和空间。
  3. 支持几乎所有类型的对象：Kryo能够序列化和反序列化几乎所有Java对象，包括自定义类、集合类、枚举、接口等。
  4. 高度可配置：Kryo提供了许多配置选项，可以根据需求进行定制，例如注册特定类、设置序列化的策略等。
  5. 跨平台支持：Kryo可以与其他编程语言的序列化库进行互操作，因此在多语言环境下也能很好地工作。
  6. 序列化版本兼容性：Kryo允许在序列化和反序列化过程中进行版本控制，以支持对象结构的演化和升级。

虽然Kryo在性能和紧凑性方面具有很大优势，但需要注意以下几点：

  1. Kryo是一个第三方库，需要将其添加到项目的依赖中。
  2. 由于Kryo使用自定义的序列化格式，序列化的数据可能不适合直接读取或编辑。

Netty一些组件

EventLoopGroup

EventLoopGroup是一个处理I/O操作的线程池。它负责处理传入和传出数据的事件，例如接收连接、读取数据、写入数据等。EventLoopGroup包含一个或多个EventLoop，每个EventLoop都负责处理一个或多个Channel的事件。其中，EventLoop通过循环方式处理事件，因此得名。
